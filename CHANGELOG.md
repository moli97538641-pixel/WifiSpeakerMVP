# 更新日志 / Changelog

本项目用于让 Android / Windows 设备通过同一个 Wi-Fi / 热点网络连接：一台设备作为发送端采集并推送本机播放音频，一台或多台设备作为接收端播放音频。

---

## v0.3.6

### 更新内容

- 基于 v0.3.5 继续优化一对多播放。
- 修复多个接收端连接时，个别接收端延迟明显偏大的问题。
- 修复低延迟策略过于激进时导致声音断断续续的问题。
- Android / Windows 发送端在开始采集前会短暂等待多个接收端完成初始连接。
- Android 接收端增加约 180ms 初始预缓冲，然后再启动 AudioTrack 播放。
- Windows 接收端增加约 180ms 初始预缓冲，然后再启动 WaveOut 播放。
- 接收端采用更保守的缓冲控制：正常情况下优先保证连续播放，缓冲明显积压时才丢帧追同步。
- 发送端每个接收端继续使用独立队列，队列满时只丢最旧帧，不清空整个队列。
- Windows 图形界面改为可滚动布局，修复内容变多后 UI 重叠的问题。
- 版本号更新为 0.3.6。

### 说明

此版本目标是让多个接收端的播放更稳定、更接近同步。它不是严格的 PTP/NTP 级时钟同步，但比 v0.3.5 更适合一对多实际使用。

## v0.3.5

### 更新内容

- 新增 Android / Windows 跨平台一对多推送。
- Android 可作为发送端或接收端。
- Windows x64 可作为发送端或接收端。
- Windows 端提供图形界面。
- 发送端可以搜索接收端设备并勾选多个接收端。
- 手动输入 IP 时支持多个地址，可用逗号、空格或换行分隔。
- 发送端启动后会同时向所有已选择的接收端推送音频。
- 每个接收端使用独立发送线程和小队列，避免单个接收端卡顿时拖慢全部接收端。
- 接收端音量滑条会同步控制所有已选择接收端的 App 内部播放增益。
- 音频流使用 10ms PCM16 stereo 帧，每帧包含 sequence 和 presentationTimeNs。
- 新增 Windows 构建工作流。

### 说明

此版本是一对多和 Windows 图形端的第一版。同步策略仍偏基础，部分设备在一对多时可能出现延迟不一致或声音断续，因此建议使用 v0.3.6 或更高版本。

## v0.3.4

### 更新内容

- 发送端界面新增“接收端音量”滑条。
- 发送端可以通过 App 内控制命令调节接收端播放音量。
- 音量控制只作用于接收端 App 内部 `AudioTrack` 的播放增益。
- 不改变发送端系统音量。
- 不改变接收端系统媒体音量。
- 新增独立音量控制端口 `45779`，避免和音频数据流混在同一连接中。
- 接收端状态页显示当前接收端应用内音量。

## v0.1.0

### 初始实验版

- 实现 Wi-Fi 音频推送的最小原型。
- 同一个 App 支持接收端和发送端两种角色。
- 接收端通过 TCP Server 等待连接。
- 发送端手动输入接收端 IP 地址。
- 使用 48kHz / 16bit / stereo PCM 音频格式进行传输。
- 接收端使用 `AudioTrack` 播放收到的 PCM 音频。
- 配置 GitHub Actions，用于在线构建 debug APK。

### 已知问题

- 发送端缺少 `RECORD_AUDIO` 权限。
- 部分设备上会出现授权后仍然无法采集系统播放音频的问题。
- 此版本属于早期实验版，不建议作为可用版本发布。

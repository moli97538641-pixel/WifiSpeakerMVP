# 更新日志 / Changelog

本项目用于让两台 Android 设备通过同一个 Wi-Fi 连接：一台设备作为接收端播放音频，另一台设备作为发送端采集并推送本机播放音频。

---

## v0.3.2

### 更新内容

- 重新修复状态栏 / 通知栏覆盖内容的问题。
- 针对 Android 15 / targetSdk 35 的 edge-to-edge 行为，改为主动读取 `WindowInsets`。
- 给每个界面的内容根布局增加顶部系统栏安全边距，避免标题和按钮被状态栏遮挡。
- 给内容根布局增加底部系统栏安全边距，避免底部内容被导航栏遮挡。
- 新增 `CHANGELOG.md`，从本版本开始维护历史更新记录。
- 更新版本号为 `0.3.2`，`versionCode` 更新为 `6`。

### 说明

`v0.3.1` 只尝试取消 fullscreen 和设置系统栏颜色，但在部分 Android 15 / targetSdk 35 设备上仍会被系统 edge-to-edge 行为影响，导致内容仍然绘制到状态栏下方。`v0.3.2` 改为使用运行时 Insets 适配，属于更可靠的修复方式。

## v0.3.0

### 更新内容

- 主界面新增角色选择：
  - 当前设备作为接收端。
  - 当前设备作为发送端。
- 接收端和发送端拆分为独立界面。
- 发送端搜索接收端设备时，使用列表展示搜索结果。
- 点击搜索列表中的设备后，自动选择目标设备。
- 保留手动输入 IP 的能力。
- 推送 / 接收按钮会根据当前运行状态自动变化：
  - 未启动时显示启动按钮。
  - 运行中时显示停止按钮。
  - 未选择目标设备时禁用推送按钮。
  - 推送中禁用搜索和 IP 编辑，避免误操作。
- 界面文案改为中性设备描述，不再绑定具体设备类型。
- 更新版本号为 `0.3.0`。

### 说明

此版本主要优化交互流程，让 App 更适合在任意两台 Android 设备之间使用。

## v0.2.0

### 更新内容

- 新增接收端设备自动发现功能。
- 发送端可以搜索同一 Wi-Fi 下的接收端设备。
- 新增断线自动重连能力。
- 当 Wi-Fi 短暂断开、接收端重启或连接异常时，发送端会尝试重新连接。
- 保留手动输入 IP 的连接方式。
- 接收端状态显示更加清楚：
  - 未启动。
  - 等待连接。
  - 已连接。
- 发送端状态显示更加清楚：
  - 未启动。
  - 搜索中。
  - 等待连接 / 重连中。
  - 推送中。

### 说明

此版本主要提升连接便利性和网络波动下的可用性。

## v0.1.1

### 修复内容

- 修复发送端缺少 `RECORD_AUDIO` 权限的问题。
- 修复部分 Android 设备上授权系统音频采集后仍然无法采集声音的问题。
- 修复“能连接但没有声音”的核心问题。
- 完善 AndroidManifest 权限声明。

### 说明

这是第一个可以正常验证 Wi-Fi 音频推送链路的版本。

核心链路为：

```text
发送端采集系统播放音频 -> Wi-Fi TCP 传输 -> 接收端播放音频
```

## v0.1.0

### 初始实验版

- 实现 Wi-Fi 音频推送的最小原型。
- 同一个 App 支持两种角色：
  - 接收端。
  - 发送端。
- 接收端通过 TCP Server 等待连接。
- 发送端手动输入接收端 IP 地址。
- 使用 48kHz / 16bit / stereo PCM 音频格式进行传输。
- 接收端使用 `AudioTrack` 播放收到的 PCM 音频。
- 配置 GitHub Actions，用于在线构建 debug APK。

### 已知问题

- 发送端缺少 `RECORD_AUDIO` 权限。
- 部分设备上会出现授权后仍然无法采集系统播放音频的问题。
- 可能表现为：
  - App 能安装。
  - 接收端能启动。
  - 发送端能点击推送。
  - 系统弹出采集授权。
  - 但接收端没有声音。

### 说明

此版本属于早期实验版，不建议作为可用版本发布。  
如需测试基础功能，请使用 `v0.1.1` 或更高版本。

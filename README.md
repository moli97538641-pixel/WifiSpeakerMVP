# WiFi Speaker MVP

让两台 Android 设备通过同一个 Wi-Fi 连接：一台作为接收端播放音频，另一台作为发送端采集并推送本机播放音频。

## v0.3.0 新增

- 主界面先选择设备角色：接收端 / 发送端。
- 接收端和发送端拆成独立界面，操作更清楚。
- 搜索接收端时展示设备列表，可点击某一台设备作为目标。
- 启动/停止按钮会根据当前状态自动变化：
  - 接收端：启动接收端 / 停止接收端。
  - 发送端：启动推送 / 停止推送。
- 推送运行中会禁用 IP 输入框和搜索按钮，避免误操作。
- 保留 v0.2.0 的自动发现和断线重连能力。

## 工作方式

```text
发送端 Android 10+
  -> MediaProjection 授权
  -> AudioPlaybackCaptureConfiguration 采集系统播放音频
  -> AudioRecord 得到 48kHz / 16bit / stereo PCM
  -> 局域网 TCP 推送

接收端设备
  -> UDP 45778 响应自动发现
  -> TCP 45777 接收 PCM
  -> AudioTrack 流式播放
```

## 使用步骤

### 接收端设备

1. 两台设备连同一个 Wi-Fi。
2. 接收端设备打开 App。
3. 主界面选择“这台设备作为接收端 / 播放音频”。
4. 点“启动接收端”。
5. 保持接收端运行，等待发送端连接。

### 发送端设备

1. 发送端设备打开 App。
2. 主界面选择“这台设备作为发送端 / 推送音频”。
3. 点“搜索接收端设备”。
4. 从搜索结果列表中点击目标接收端设备。
5. 点“启动推送到 xxx.xxx.xxx.xxx”。
6. 允许录音权限和系统投屏/录制授权。
7. 在发送端设备上播放普通音乐或视频，声音会从接收端设备播放。

如果自动搜索失败，也可以手动输入接收端设备界面显示的 IP。

## 注意事项

- 发送端需要 Android 10 / API 29 或更高版本。
- 某些 App 会禁止被系统音频采集，DRM、通话、受保护内容通常采不到。
- 自动发现依赖同一局域网 UDP 广播；访客网络、AP 隔离、部分路由器防火墙会导致搜不到。
- 当前版本仍是 MVP，音频未压缩 PCM 直传，音质优先，带宽约 1.5 Mbps。

## 不用 Android Studio 打包

本项目包含 GitHub Actions 配置。上传到 GitHub 仓库后，进入 Actions，运行 `Build Android APK`，构建成功后从 Artifacts 下载 `app-debug.apk`。

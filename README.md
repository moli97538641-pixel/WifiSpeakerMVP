# WifiSpeakerMVP v0.3.4

两台 Android 设备通过同一个 Wi-Fi 连接：一台作为接收端播放音频，另一台作为发送端采集并推送本机播放音频。

## v0.3.4 更新

- 发送端界面新增“接收端音量”滑条。
- 发送端可以在 App 内控制接收端播放音量。
- 音量控制只作用于接收端 App 内部的 `AudioTrack` 播放增益。
- 不改变发送端系统音量。
- 不改变接收端系统媒体音量。
- 新增独立音量控制端口 `45779`。
- 保留 `v0.3.3` 的保守系统栏处理和历史更新记录。

## 当前功能

- 主界面先选择设备角色：接收端 / 发送端。
- 接收端和发送端拆成独立界面。
- 搜索接收端时展示设备列表，可点击某一台设备作为目标。
- 启动 / 停止按钮会根据当前状态自动变化：
  - 接收端：启动接收端 / 停止接收端。
  - 发送端：启动推送 / 停止推送。
- 发送端可调节接收端应用内播放音量。
- 推送运行中会禁用 IP 输入框和搜索按钮，避免误操作。
- 保留自动发现和断线重连能力。
- 保留手动输入 IP 的能力。

## 工作方式

```text
发送端 Android 10+
  -> MediaProjection 授权
  -> AudioPlaybackCaptureConfiguration 采集系统播放音频
  -> AudioRecord 得到 48kHz / 16bit / stereo PCM
  -> 局域网 TCP 推送音频
  -> TCP 45779 发送接收端音量控制命令

接收端设备
  -> UDP 45778 响应自动发现
  -> TCP 45777 接收 PCM
  -> AudioTrack 流式播放
  -> 对 AudioTrack 应用 App 内部播放增益
```

## 使用步骤

### 接收端设备

1. 两台设备连同一个 Wi-Fi。
2. 接收端设备打开 App。
3. 主界面选择“这台设备作为接收端 / 播放音频”。
4. 点“启动接收端”。
5. 保持接收端运行，等待发送端连接和音量控制。

### 发送端设备

1. 发送端设备打开 App。
2. 主界面选择“这台设备作为发送端 / 推送音频”。
3. 点“搜索接收端设备”。
4. 从搜索结果列表中点击目标接收端设备。
5. 使用“接收端音量”滑条调整接收端播放音量。
6. 点“启动推送到 xxx.xxx.xxx.xxx”。
7. 允许录音权限和系统投屏 / 录制授权。
8. 在发送端设备上播放普通音乐或视频，声音会从接收端设备播放。

如果自动搜索失败，也可以手动输入接收端设备界面显示的 IP。

## 注意事项

- 发送端需要 Android 10 / API 29 或更高版本。
- Android 13 及以上需要允许通知权限，否则前台服务通知可能不会显示在通知栏。
- 接收端音量滑条只控制接收端 App 内部播放增益，不会改变发送端系统音量。
- 某些 App 会禁止被系统音频采集，DRM、通话、受保护内容通常采不到。
- 自动发现依赖同一局域网 UDP 广播；访客网络、AP 隔离、部分路由器防火墙会导致搜不到。
- 当前版本仍是 MVP，音频未压缩 PCM 直传，音质优先，带宽约 1.5 Mbps。
